# Jira Revision Pack v1

**Dự án:** PharmaAssist AI Intelligence
**Loại tài liệu:** Jira Revision Plan / Jira Backlog Rewrite
**Trạng thái:** Draft v1
**Ngôn ngữ chính:** Tiếng Việt
**Thuật ngữ kỹ thuật:** Giữ tiếng Anh khi cần

---

## 1. Mục đích

Tài liệu này dùng để revise Jira theo baseline mới của PharmaAssist AI Intelligence sau khi đã có:

1. Requirement ID Registry.
2. Use Case ID Registry.
3. Test Case ID Registry.
4. Traceability Matrix tổng.

Mục tiêu của Jira revision:

1. Xóa hoặc rewrite các Jira item cũ lệch baseline.
2. Chuẩn hóa Components.
3. Chuẩn hóa Epics.
4. Tạo lại Stories theo MVP critical path.
5. Gắn Requirement ID, Use Case ID, Test Case ID và Trace ID vào từng Story.
6. Bảo đảm Jira là source of truth cho Epic / Story / Sprint / Status.
7. Chuẩn bị cho Gap Analysis giữa code hiện tại và documentation.

---

## 2. Nguyên tắc revise Jira

### 2.1. Không quay lại baseline cũ

Jira item cũ phải được rewrite hoặc xóa nếu chứa các nội dung sau:

| Nội dung cũ                                | Hành động                                                  |
| ------------------------------------------ | ---------------------------------------------------------- |
| Custom JWT / username-password auth        | Rewrite sang Supabase Auth                                 |
| Lưu `password_hash` trong PostgreSQL       | Xóa hoặc rewrite                                           |
| Aggregate inventory là source of truth     | Rewrite sang MedicineBatch source of truth                 |
| Direct edit inventory quantity             | Rewrite sang Inventory Adjustment workflow                 |
| Medicine-level interaction rule chính thức | Rewrite sang ActiveIngredient-level DrugInteraction Rule   |
| MockAI-only MVP                            | Rewrite sang Google AI primary + MockAI fallback           |
| MockGraph-only MVP                         | Rewrite sang Neo4j projection thật                         |
| Neo4j là source of truth                   | Rewrite sang PostgreSQL source of truth                    |
| `/orders/{id}/pay` là command chính        | Rewrite sang Checkout API                                  |
| Direct invoice creation normal flow        | Rewrite sang invoice generated by checkout                 |
| Full Customer Management là MVP blocker    | Rewrite sang walk-in/anonymous customer                    |
| Full 100-table MVP schema                  | Rewrite sang MVP core subset                               |
| ProductVariant làm MVP sales key           | Rewrite sang `medicine_id`                                 |
| Raw Cypher API/UI cho Staff                | Xóa hoặc rewrite sang Graph-RAG query template             |
| DrugGroup từ category data trong MVP       | Xóa hoặc chuyển Future                                     |
| Symptom/Condition/RedFlag graph MVP        | Xóa hoặc chuyển Future                                     |
| riskScore ảnh hưởng severity/checkout      | Xóa                                                        |
| Docker là official setup path              | Rewrite sang local Node.js + cloud Supabase + Neo4j AuraDB |
| Destructive tests against demo DB          | Xóa hoặc rewrite sang safe local isolation                 |

## 5. Jira Custom Fields / Labels đề xuất

Nếu Jira cho phép custom fields, nên thêm các field sau:

| Field           | Ví dụ                                                                   |
| --------------- | ----------------------------------------------------------------------- |
| Requirement IDs | `FR-CHK-01, FR-CHK-03`                                                  |
| Use Case IDs    | `UC-CHK-01`                                                             |
| Test Case IDs   | `TC-CHK-01, TC-CHK-03`                                                  |
| Trace IDs       | `TR-CHK-01`                                                             |
| Scope           | `MVP`, `Should-have`, `Future`, `Out of Scope`                          |
| Priority        | `P0`, `P1`, `P2`, `P3`                                                  |
| Module          | `Checkout`, `AI Guardrail`, `Graph Sync`                                |
| Baseline Status | `New Baseline`, `Rewrite Required`, `Remove Old Scope`, `Gap Candidate` |

Nếu Jira không có custom fields, đưa các thông tin này vào phần Description của từng Story theo template bên dưới.

# 10. Definition of Ready

1. Epic.
2. Component.
3. Scope.
4. Priority.
5. Requirement ID.
6. Use Case ID.
7. Test Case ID.
8. Trace ID.
9. Acceptance Criteria.
10. API/UI/Database impact.
11. Out-of-scope guard nếu có.
12. Owner hoặc assignee tạm thời.
13. Estimate tạm thời.

# 11. Definition of Done

Một Jira Story được xem là Done khi:

1. Code đã merge vào `develop`.
2. Lint pass.
3. Type check pass.
4. Unit/integration test liên quan pass.
5. Prisma validation pass nếu đụng database.
6. Migration check pass nếu đụng schema.
7. Frontend build pass nếu đụng UI.
8. Backend build pass nếu đụng backend.
9. Test Case ID liên quan đã được cập nhật trạng thái.
10. Traceability Matrix cập nhật status.
11. Không vi phạm Negative Scope Control.
12. PR có link Jira issue.
13. Acceptance Criteria được verify.
