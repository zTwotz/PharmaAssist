# Kế hoạch tạo lại bộ Jira Files cho PharmaAssist AI Intelligence

## 1. Mục tiêu

Tạo lại 6 file Jira chính thức:

1. `1_Components.md`
2. `2_Epic.md`
3. `3_Stories.md`
4. `4_Task.md`
5. `5_Sprint.md`
6. `branch-on-jira.md`

Các file mới phải đủ chi tiết để:

1. Đưa trực tiếp vào Jira.
2. Cho AI agent đọc và code theo đúng scope.
3. Trace được từ Epic → Story → Task → Requirement → Use Case → Test Case → API → UI → Database.
4. Bao gồm cả MVP và phần mở rộng nâng cao về AI, Graph, Neo4j, Graph-RAG, demo, testing, CI.
5. Không quay lại baseline cũ.

---

## 2. Nguyên tắc tạo bộ Jira mới

### 2.1. Không dùng nguyên bộ Jira cũ

Bộ Jira cũ chỉ dùng làm reference về format, không dùng làm source of truth.

Các nội dung phải loại bỏ hoặc rewrite:

1. Custom JWT/password auth.
2. `password_hash` trong PostgreSQL.
3. Aggregate inventory làm source of truth.
4. `inventory` hoặc `stock_batches` mơ hồ thay cho `MedicineBatch`.
5. Direct edit inventory quantity.
6. Payment API tách rời checkout.
7. Direct invoice creation.
8. Medicine-level interaction rule official.
9. MockAI-only MVP.
10. Mock Knowledge Graph thay cho Neo4j thật.
11. Forecast/Supabase Realtime/Notification bị đưa thành MVP blocker.
12. Docker là official setup path.
13. Destructive tests against demo DB.

### 2.2. Jira mới phải chia scope rõ

Mỗi Epic/Story/Task sẽ có field:

```text
Scope:
- MVP
- Should-have
- Future / Commercial Expansion
- Out of Scope Guard
```

### 2.3. Jira mới phải có traceability

Mỗi Story và Task nên có:

```text
Requirement IDs:
Use Case IDs:
Test Case IDs:
Trace IDs:
Related API:
Related UI:
Related Database:
```

### 2.4. Jira mới phải đủ chi tiết cho AI agent

Mỗi Story/Task không chỉ có title. Phải có:

1. Context.
2. User story hoặc technical story.
3. Acceptance Criteria.
4. API details.
5. UI details.
6. Database details.
7. Validation rules.
8. Authorization rules.
9. Error cases.
10. Test coverage.
11. Out-of-scope guardrails.
12. Notes for AI agent.

# 5. Quy tắc chi tiết để AI agent code đúng

Mỗi Story/Task sẽ có phần riêng:

## 5.1. AI Agent Context

```text
You are implementing this Jira issue for PharmaAssist AI Intelligence.
Follow the official baseline:
- Next.js frontend
- NestJS backend
- Prisma ORM
- Supabase Auth
- PostgreSQL source of truth
- Neo4j projection only
- Google AI primary, MockAI fallback
- MedicineBatch inventory source of truth
```

## 5.2. AI Agent Must Not

```text
Do not implement custom JWT auth.
Do not store password_hash.
Do not use aggregate inventory as source of truth.
Do not expose raw Cypher to Staff.
Do not use Neo4j to decide checkout.
Do not create direct payment endpoint as main flow.
Do not create invoice outside checkout transaction.
Do not let AI diagnose, prescribe, or give dosage advice.
```

## 5.3. AI Agent Must Implement

```text
Implement API with authorization.
Implement validation.
Implement database schema/migration if needed.
Implement service logic.
Implement UI states if frontend task.
Implement tests.
Update traceability note.
Respect MVP/Should-have/Future scope.
```

# 6. Definition of Done chung cho tất cả Jira item

Một Story/Task chỉ được xem là Done khi:

1. Code đã đúng baseline.
2. Không vi phạm Negative Scope Guard.
3. API có validation/error handling.
4. Authorization được enforce ở backend.
5. UI không expose chức năng sai quyền.
