# Requirement ID Registry v1

**Dự án:** PharmaAssist AI Intelligence
**Loại tài liệu:** Requirement ID Registry
**Trạng thái:** Draft v1
**Ngôn ngữ chính:** Tiếng Việt
**Thuật ngữ kỹ thuật:** Giữ tiếng Anh khi cần

---

## 1. Mục đích

Tài liệu này định nghĩa danh sách **Requirement ID** chính thức để dùng làm nền cho các bước tiếp theo:

1. Use Case ID Registry.
2. Test Case ID Registry.
3. Traceability Matrix tổng.
4. Jira Components / Epics / Stories revision.
5. Gap Analysis giữa code hiện tại và documentation.

Requirement ID Registry không thay thế SRS, Use Case Specification, API Specification, Database Design hoặc UI/UX Specification. Tài liệu này chỉ đóng vai trò **bảng mã hóa requirement** để bảo đảm mọi artifact sau này có thể trace về cùng một baseline.

---

## 2. Quy ước đặt mã Requirement

### 2.1. Functional Requirement ID

Format:

```text
FR-[MODULE]-[NUMBER]
```

Ví dụ:

```text
FR-AUTH-01
FR-RBAC-01
FR-MED-01
FR-CHK-01
FR-AIG-01
FR-GSY-01
```

### 2.2. Module Prefix

| Prefix | Module / Area                      | Ghi chú                                                   |
| ------ | ---------------------------------- | --------------------------------------------------------- |
| AUTH   | Authentication                     | Supabase Auth, login, logout, first-login password change |
| RBAC   | Roles, Permissions & Authorization | Multi-role RBAC, permission, ownership scope              |
| USER   | User / Staff Account Management    | User profile, staff account, role assignment              |
| MED    | Medicine Management                | Medicine master data                                      |
| ACT    | ActiveIngredient Management        | ActiveIngredient và medicine ingredient mapping           |
| SUP    | Supplier Management                | Supplier CRUD/deactivation                                |
| BAT    | MedicineBatch & Inventory Summary  | Batch-level inventory source of truth                     |
| STI    | Stock Import                       | Nhập kho, confirm import, merge/reject batch              |
| ADJ    | Inventory Adjustment               | Điều chỉnh tồn kho có reason/audit                        |
| POS    | POS Draft Order                    | Lập đơn bán hàng ở trạng thái DRAFT                       |
| DRG    | DrugInteraction Rule               | ActiveIngredient-level interaction rule                   |
| ALT    | InteractionAlert                   | Persist alert, HIGH acknowledgement/note                  |
| CHK    | Checkout                           | Transaction hoàn tất order/payment/invoice                |
| PAY    | Payment                            | SUCCESS/FAILED payment attempts                           |
| INV    | Invoice                            | Invoice generated by checkout                             |
| AIC    | AI Copilot                         | AI explanation, note draft, follow-up questions           |
| AIG    | AI Guardrail                       | Input/output guardrail, blocking unsafe requests          |
| AIA    | AI Audit                           | AI Audit Log, provider/prompt metadata                    |
| GSY    | Graph Sync                         | PostgreSQL → Neo4j projection                             |
| GRG    | Graph-RAG                          | Graph-RAG query, provenance, fallback                     |
| RPT    | Reports                            | Revenue, top medicines, inventory report                  |
| SET    | System Settings                    | Near-expiry threshold, basic configuration                |
| DMO    | Demo Data & Demo Reset             | Seed data, demo reset workflow                            |
| AUD    | System Audit                       | Backend audit logging                                     |
| DEV    | DevOps / CI / Setup                | CI gates, setup constraints                               |
| TST    | Testing                            | Test strategy, smoke test, test safety                    |

## 3. Scope Classification

| Scope                         | Ý nghĩa                                                          |
| ----------------------------- | ---------------------------------------------------------------- |
| MVP                           | Bắt buộc cho phiên bản demo/chính thức của đồ án                 |
| Should-have                   | Nên có nếu đủ thời gian, không chặn MVP                          |
| Future / Commercial Expansion | Dành cho mở rộng thương mại hoặc phiên bản sau                   |
| Out of Scope                  | Không làm trong MVP và không nên đưa vào implementation hiện tại |

---

## 4. Priority Classification

| Priority | Ý nghĩa                                                     |
| -------- | ----------------------------------------------------------- |
| P0       | Critical path, nếu thiếu sẽ ảnh hưởng demo/MVP nghiêm trọng |
| P1       | Core requirement, cần có để hệ thống đầy đủ                 |
| P2       | Supporting requirement, hỗ trợ vận hành/demo                |
| P3       | Enhancement hoặc future-facing requirement                  |

| Checkout             | UC-CHK-01              |
| AI Copilot           | UC-AIC-01              |
| Graph-RAG            | UC-GRG-01              |
